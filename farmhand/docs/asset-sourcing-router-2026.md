# Asset-Sourcing Router (2026)

How the [[CMO]] decides, **per slide / per post**, which tool makes each visual —
so it never burns a Higgsfield credit on something a chart or a stock photo
could do, and never cheapens a hero with a generic stock shot. Pairs with
`content-engine-spec-2026.md` (archetypes) + `higgsfield-prompting-playbook-2026.md`.

## The three sources
| Source | What it is | Cost | Use for |
|---|---|---|---|
| **RENDERED** | In-app DESERT GRID design functions — charts, tables, big-number cards, diagrams, maps, timelines, typographic slides | free · always on-brand | all data/text visuals |
| **STOCK** | Pexels / Unsplash / Pixabay (live connectors) | free | atmospheric/texture backgrounds, treated so they don't read generic |
| **HIGGSFIELD** | Soul 2.0 (custom photoreal stills) + Cinema Studio / Seedance (video) | credits | real specific scenes (field proof) + **all** reels/video |

## The decision (engine runs this per slide)
```
1. Is the slide a chart, table, big number, diagram, map, timeline, quote,
   myth-bust, or pure text?              -> RENDERED  (free, always — the
                                            majority of carousel slides)
2. Is it a REEL or any video?            -> HIGGSFIELD (Cinema Studio / Seedance).
                                            Never stock video. Non-negotiable.
3. Does it need a REAL, SPECIFIC scene
   as proof or authenticity — a rooftop,
   installer, meter, bill on a table, a
   battery wall, an AZ home/neighborhood?-> HIGGSFIELD Soul  (or a real phone
                                            photo Taylor shot — even better).
                                            NEVER stock here (§A09 rule).
4. Does it only need atmosphere/texture
   behind a scrim, where a non-specific
   shot is fine (sky, heat haze, desert
   horizon, abstract)?                   -> STOCK, treated (see below)
5. Hero/cover and unsure stock vs custom?-> HIGGSFIELD. For the hero, credibility
                                            beats saving a credit.
```

## Per-archetype default source
| Archetype | Default | Notes |
|---|---|---|
| A01 hero number · A02 ranked bar · A03 line · A04 small-multiples · A05 map · A07 statement · A08 bullets · A10 myth-bust · A11 timeline · A12 quote · A14 scorecard · A15 diagram · A16 closer | **RENDERED** | never needs a photo; this is most of a carousel |
| A06 editorial cover | RENDERED **or** HIGGSFIELD | typographic cover for data topics; photo cover when a scene hooks harder |
| A09 photo + scrim | **HIGGSFIELD** (or real photo) | field proof — spec says **zero stock** here |
| Any **Reel** | **HIGGSFIELD** | Cinema Studio (single-shot from a still) / Seedance (multi-shot). Real phone footage where Taylor can shoot it. |

## Using stock without looking generic (the careful rules)
Stock is a **graphic element, not a photograph of a real moment.** Never present
a stock shot as documentation of an actual job/home/person.
- **Treat it:** DESERT GRID `BG-6` duotone (map to NIGHT→PAPER, one accent object in color) or a heavy NIGHT scrim, so it reads as designed, not stock. Never drop raw stock behind text.
- **Only atmosphere:** sky, heat shimmer, desert horizon, asphalt, abstract texture, wide unpeopled landscape. Never a person's face, never a "smiling family / handshake / lightbulb" cliché.
- **Never the hero of a proof/field post.** Stock supports; it never claims.
- **Never two stock-looking slides adjacent** (grid test).
- If a stock search only returns generic/cliché results → fall back to RENDERED or a Higgsfield custom.

## Why this saves money and looks better
- Most carousel slides are data → **RENDERED, free, perfectly on-brand.** Higgsfield is spent only where a real scene earns it.
- The credibility lane (rooftops, installers, bills, batteries — the "I was there" proof) is exactly where custom Higgsfield/real photos beat stock, so that's where credits go.
- Reels always custom (Higgsfield) — that's the reach engine and it must look premium.

## Guardrails
- **Credit discipline:** no Higgsfield spend on anything RENDERED can do. Batch/queue Higgsfield jobs; never auto-spend before a brief is `approved`.
- **Authenticity (inherited):** only a REAL photo may be captioned as a specific real job/meeting/install. A Higgsfield image is generic illustration; stock is a graphic. Never blur that line.
- **The eyeball gate:** every generated/sourced asset passes the Higgsfield playbook QA (believable? on-palette? no problematic text?) before it can go `visuals-planned`.
